## 0. 项目概述与技术栈

**admin-frontend**：基于 Vue 3 + TypeScript + Vite 构建的前端管理系统，包含后台管理功能（RBAC、系统支撑、聊天、日志等）和公共展示功能（博客、视频）。

**核心技术栈**：
- Vue 3.4+（Composition API）、Vite 5.0+、TypeScript 5.3+
- Pinia 2.1+（状态管理）、Vue Router 4.2+
- Axios 1.6+（HTTP）、Element Plus 2.4+（UI）、vue-i18n 9.9+（国际化）
- video.js 8.23+（视频播放）

**目录结构要点**（详见 `.cursor/rules/20-frontend.mdc`）：
- `src/api/generated/`：goctl 从后端 `.api` 生成的接口代码（禁止手改）
- `src/api/*.ts`：对 generated 的二次封装
- `src/components/common/D2Table.vue`：统一表格组件（列表+分页+详情/编辑+新增）
- `src/stores/dict.ts`：全局字典状态；`src/composables/useDictOptions.ts`：字典选项 composable
- `src/views/system/`、`src/views/blog/`、`src/views/video/`、`src/views/chatroom/`：后台业务页面
- `src/views/public/`：公共（无需登录）展示页面
- `src/styles/public-list.scss`、`src/styles/public-detail.scss`：公共页面通用样式模板

**前后端协同**：后端定义 `.api` → goctl 生成后端 → goctl 生成前端 TS（`scripts/generate-ts.sh`）→ 前端二次封装 → 页面使用。

**部署**：`pnpm dev`（本地代理 `/api` → `localhost:20000`）；`pnpm build` 输出 `dist/`，部署路径 `/admin/`，生产环境通过 Nginx 代理网关。

开发流程、API/字典/组件使用规范已迁移至 `.cursor/rules/00-workflow.mdc`、`.cursor/rules/20-frontend.mdc`、`.cursor/rules/21-public-pages.mdc`，请查阅规则文件。

## 2. 已完成功能（历史记录）

- 初始化项目骨架（Vite + Vue3 + TS + Pinia + Router + Element Plus），登录、基础布局、菜单导航、Token 拦截。
- **阶段三 RBAC**：角色管理（`RoleList.vue`）、权限管理（`PermissionList.vue`）、部门管理（`DepartmentList.vue`，树形）、菜单管理（`MenuList.vue`，树形）、用户管理（`UserList.vue`），均为列表+分页+新增/编辑/删除。
- **阶段四 系统支撑**：系统配置（`ConfigList.vue`，支持刷新缓存）、数据字典类型/项管理（`DictTypeList.vue`/`DictItemList.vue`）、文件管理（`FileList.vue`，含上传/下载），均已接入 D2Table。
- **demo 功能页面**：`DemoList.vue` 作为开发流程示例，由脚本自动生成骨架，展示标准 CRUD + 权限绑定写法。
- **阶段七 聊天功能**：会话列表（`ChatList.vue`）、消息列表（`ChatMessageList.vue`）、群组管理（`ChatGroupList.vue`，含成员管理），WebSocket 实时收发与在线状态。
- **阶段八 公告与通知**：公告管理（`NoticeList.vue`，支持定时发布）、消息通知（`NotificationList.vue`，已读/全部已读/清除/删除）。
- **阶段九 性能日志**：`PerformanceLogList.vue`，支持按方法/路径/慢接口/状态码/时间范围筛选。
- **阶段十 视频管理**：`VideoList.vue`（列表管理）+ `VideoPlayer.vue`（video.js 播放器，支持 m3u8/mp4，智能代理播放：先直连失败后走代理）。
- **阶段十一 异步任务**：`TaskList.vue`（任务详情、参数/结果 JSON、取消任务）、`TaskFloatBall.vue`（浮动任务球，实时状态、完成任务可直接下载）；操作/登录/审计/SDK调用/性能日志的导出功能统一改为创建异步任务，不再直接触发 blob 下载。
- **SDK 管理（首期）**：SDK API Key 管理（列表/新增/编辑/删除、IP 白名单、过期时间戳），权限绑定 `sdk_key:*`。
- **数据统计**：`MetricStats.vue`，按 module/bizId/day 查询 PV/UV/VV/IP，数值卡片 + 柱状对比图。
- **博客功能模块**（详见 §6.1）：标签/文章/审核管理 + 公共博客列表/详情页 + 打点上报。
- **博客扩展功能**（详见 §6.2）：友情链接、社交信息、文章置顶、标签筛选。
- **Public 页面小程序风格改造**（详见 §6.3）：博客、视频公共页统一视觉与交互，含移动端滚动位置恢复。

## 4. 技术决策记录

- 2025-12-24：前端使用 goctl TS 生成 + Axios 二次封装；对生成代码中的 path param 问题在自定义 API 层修正（不修改 generated）。
- 2025-12-24：权限指令 `v-permission` 基于 Pinia 权限列表与 `permissions.includes('*')`；路由 `meta.permission` 校验不通过跳转 403。
- 2025-12-24：权限/菜单本地缓存 TTL 5 分钟；动态路由由菜单树驱动并开启 KeepAlive；主题切换通过 `data-theme`。
- 2026-01-13：导出功能统一走「异步任务 + 文件管理」模式：导出按钮只创建异步任务，不直接请求 blob 下载；任务完成后通过任务详情结果 JSON（如 `fileUrl`）获取下载链接，由文件管理模块统一承载下载逻辑。
- 曾尝试将 Public 页面迁移到独立的 admin-nuxt（Nuxt3 SSR）项目，已于后续版本完全回滚合并回 admin-frontend，不再重复此方向。
- 博客「标准博客风格」改造（顶部导航栏、左侧分类导航、右侧 TOC 目录、阅读时间/字数统计等）曾完成详细设计评审，但未实施；当前 Public 页面维持「小程序风格」（见 `.cursor/rules/21-public-pages.mdc`），该方向搁置，不再维护详细设计稿。

## 7. DDD-lite 重构后前端联调（2026-07-07）

后端 DDD-lite 重构**未改变 HTTP 路径**，现有 `src/api/generated/` 与 `blog.ts` 等封装层可继续用。

**联调步骤**（详见 [`docs/admin-server-ddd-smoke-test.md`](admin-server-ddd-smoke-test.md) 第三节）：

1. 后端 `go build ./...` + 本地启动验证接口
2. **用户执行** `admin-server/scripts/generate-ts.sh`（仅当 `admin.api` 有变更时需要；当前对比结论：169 函数名 / 168 路径与现网一致，可不重生成）
3. `npm install` → `npm run typecheck` → `npm run dev`
4. 走查：登录 → Blog 后台 CRUD → `/blog/` 公开页 → 异步导出任务浮球

**注意**：`package.json` 的 `api:gen` 已失效，唯一生成入口是 `generate-ts.sh`。
