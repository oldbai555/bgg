> - **现象**：`git commit` 被本仓库的 pre-commit AI 审查（Gentleman Guardian Angel）连续拦了两轮，指出的问题（squirrel 规范、README 现状描述过期、种子 SQL 路径过期）大多和本次"文档治理"这个任务主题无关，只是恰好本次 diff 改了这些文件里的一行注释/链接，审查工具对整份改动文件做了全量合规检查而不只审查 diff 本身。**根因**：这类 pre-commit 审查按"文件是否被本次提交触碰"而非"本次改动是否引入问题"来定合规范围，触碰一行注释就会把同文件里所有存量违规一起摊出来。**正确做法**：遇到这种情况不要盲目 `--no-verify` 绕过，也不要不假思索地把发现的问题全部动手修掉——先判断问题是否真实存在（本次逐条用 `grep`/`Read` 核实过，均非误报）、是否会明显扩大本次提交的改动范围和风险（尤其是种子 SQL 这种涉及数据库语义的改动），拿不准就像本次一样明确列出来问用户"要不要一并修"，得到用户"一并修复"的明确指示后再动手，不要单方面决定扩大范围。
