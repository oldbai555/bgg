**踩过的坑 —— 绝对不要再犯**：

> - **现象**：目录迁移时按精确字符串 `grep -rn "admin-server/docs/changelog"` 排查引用，一开始漏掉了两处。**根因**：`admin-server/docs/progress.md` 里用的是相对 Markdown 链接 `[docs/changelog/](changelog/README.md)`，链接目标 `changelog/README.md` 不含 `admin-server/` 前缀，精确字符串匹配搜不到。**正确做法**：迁移目录时除了 grep 完整旧路径，还要单独 grep 目录名本身（如 `"changelog/"`）和目录内文件名，排查所有相对链接，不能只信一种搜索方式。
> - **现象**：提交本次改动时，本仓库的 Gentleman Guardian Angel pre-commit 审查两次判定 FAILED，理由是"新规则要求会话结束必须写交接摘要，但本次会话自己没有新增 changelog 条目"。**根因**：容易忘记刚写完的规则从下一次提交起就对自己生效，不是只对以后的会话生效。**正确做法**：改完 `docs/changelog/` 相关规则后，当次提交本身也要照着新规则补一篇（就是本文件），"规则强制"不是写给别人看的摆设。
